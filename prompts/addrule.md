## 指令：向 Rule 文件追加新规范

在 rule 文件的适当位置插入一条新的规范条目。

### 执行步骤

1. 阅读目标 rule 文件，理解现有章节结构

2. 判断新规范属于哪个章节（由调用方描述，Agent 自行判断归属）

3. 检查是否已存在类似规范：
   - 若已存在相似条目，输出「已有类似规范：<章节名>/<规范名>」，不重复添加
   - 若不存在，在对应章节末尾插入新条目

4. 新条目格式要求：
   - 包含「必须遵循/推荐/禁止」约束标记
   - 附好坏示例各一个，示例不超过 15 行，注释中文
   - 示例若调用方已提供则直接使用，否则根据描述编写

5. 输出：插入位置（章节名）、新规范标题

### 调用方需在 prompt 中指定

- 目标文件：`rules/frontend-rule.mdc` 或 `rules/backend-rule.mdc`
- 新规范描述：如「所有 API 请求必须使用 TanStack Query，禁止裸 fetch」
- 示例代码（可选）
