## 指令：加载并遵循 Rule 文件

在开始执行任何开发任务之前，完整阅读指定的 rule 文件，后续所有代码编写严格遵循其中的规范。

### 执行步骤

1. 读取对应的 rule 文件：
   - 前端开发任务：`rules/frontend-rule.mdc`
   - 后端开发任务：`rules/backend-rule.mdc`

2. 逐章节阅读，重点关注：
   - 标记为「必须遵循」的规范点
   - 标记为「禁止」的反模式
   - 好坏示例中展示的具体差异

3. 阅读完成后在内部确认：已加载规范，后续代码输出将严格遵守。

4. 如果 rule 文件不存在（`rules/frontend-rule.mdc` 或 `rules/backend-rule.mdc`），
   输出警告：「rule 文件不存在，建议先运行 genRule 指令生成规范文件」，然后继续执行任务。

### 注意

- 不要在输出中重复 rule 文件的全部内容
- 仅在违反规范风险较高时提示具体规范条目
