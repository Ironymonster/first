## 指令：代码质量优化

分析 `backend/` 或 `frontend/` 目录下本次需求（REQ-<id>）涉及的代码，进行复用性和性能优化。

### 优化范围

通过以下方式确定本次需求涉及的文件：
```bash
git diff HEAD~1 --name-only
```
只分析和优化上述 diff 涉及的文件，不要改动无关文件。

### 优化目标

1. **复用性**：提取重复逻辑为公共函数/组件，消除冗余代码
2. **性能**：识别低效的数据库查询、不必要的重渲染、多余的网络请求
3. **清理**：删除未使用的变量、导入、死代码
4. **可读性**：必要时重构过于复杂的函数（单一职责）

### 执行规则

- 如果当前代码已经足够好，明确说明原因，不要为了优化而优化
- 每次修改后运行构建/测试验证不破坏现有功能：
  - 前端：`cd frontend && npm run build`
  - 后端：`cd backend && go test ./...`
- 优化完成后输出简洁的变更摘要（改了哪些文件，做了什么优化）
