## 指令：局部更新 Rule 文件

只修改 rule 文件中指定的章节内容，不影响其他章节。

### 执行步骤

1. 阅读目标 rule 文件，定位需要更新的章节（由调用方在 prompt 中指定）

2. 分析项目中与该章节相关的实际代码，了解当前习惯

3. 仅修改指定章节，保持其他章节完全不变，更新要求：
   - 好坏示例都要有，示例不超过 15 行
   - 如果调用方提供了示例代码，以提供的为准；否则从实际代码中提取
   - 如果该章节已有对应示例则跳过，不重复添加

4. 输出修改摘要：修改了哪个章节、增删了哪些规范点

### 调用方需在 prompt 中指定

- 目标文件：`rules/frontend-rule.mdc` 或 `rules/backend-rule.mdc`
- 章节名称：如「数据请求规范」「错误处理规范」
- 更新方向（可选）：如「补充 zod 校验示例」「移除过时的 axios 写法」
